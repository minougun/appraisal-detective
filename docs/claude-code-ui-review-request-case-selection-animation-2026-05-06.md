## 現在の到達点

`鑑定探偵` は、不動産鑑定評価を題材にしたノベルADV + 机上捜査ゲームです。

現在は以下の方向で作っています。

- タイトルから案件選択までを、Webサイトではなくゲーム画面として見せる。
- 案件選択は3枚の物理的な事件ファイルを机上に並べる。
- ファイル選択後、選ばなかったファイルはフェードアウトし、選んだファイルだけが中央に大きく残ってから案件へ進む。
- 各フェーズ冒頭はノベルADV風の会話シーンで始まる。
- 会話が終わるまで、受任面談・現地調査などの本体操作は見せない。
- 会話後の操作画面も、全て暗い机上のゲーム画面内に収める。
- 主人公の新人鑑定士、先輩鑑定士、依頼者が会話に出る。
- 現地調査では写真内の違和感をタップして証拠カードを得る。
- 証拠カードや学びカードは、読ませるカードとして短く自然にポップアップする。
- 圧力フラッシュ、証拠ボードの脈動、先輩メモ更新などは状態フィードバック用途として短く出す。

## 直近の変更点

### 1. 案件選択ファイル演出

該当箇所:

- `app.js`
  - `CASE_SELECT_TRANSITION_MS`
  - `CASE_SELECT_HOLD_MS`

  - `animateCaseSelection()`
- `styles.css`
  - `.case-file-desk.case-selecting`
  - `.case-file-selected`
  - `.case-file-dismissed`

変更内容:

- 案件ファイルをクリックすると、他のファイルがフェードアウトする。
- 選んだファイルだけが中央へ移動し、少し拡大する。
- 中央に来てから一拍置いて、ノベル会話へ遷移する。
- FirefoxではChromeより演出が短く見えたため、Firefoxのみ中央保持時間を長めにしている。
- `requestAnimationFrame()` を2回挟んでから遷移タイマーを開始し、CSS transition の初回描画が飛ばないようにしている。

直近の問題意識:

- Chromeでは自然に見える。
- Firefoxでは以前、中央表示が一瞬で消えるように見えていた。
- そのため、Firefox向けに保持時間を伸ばしたが、ブラウザ分岐が妥当か、演出として不自然でないか見てほしい。

## UIレビューで特に見てほしい論点

### A. 案件選択アニメの自然さ

確認してほしいこと:

- 事件ファイルを選んだ時、選択ファイルが中央へ出てくる演出は気持ちよいか。
- 他のファイルのフェードアウトは早すぎないか、遅すぎないか。
- 選んだファイルが中央に出てから消えるまでの「溜め」は十分か。
- Chrome / Firefox で体感差がまだ残っていないか。
- Firefox向けの長め保持が、Chromeとの差として気持ち悪くないか。
- `受験番長` のキャラ選択のような「選んだものが主役になる」演出に近づいているか。

辛口で見てほしい観点:

- まだWebのカード選択に見えていないか。
- 物理的な事件ファイルを開く演出として説得力があるか。
- 中央に寄るだけで、ゲーム的な手触りが足りないか。
- 追加するなら、紙の影、机の照明、軽いズーム、画面暗転、SE、スタンプ音など何が一番効くか。

### B. カードポップアップの自然さ

確認してほしいこと:

- 証拠取得カードのフェードインがまだ唐突ではないか。
- 表示時間は短すぎないか、長すぎないか。
- 学びカード、証拠提示カード、専門用語バーストで、表示時間の役割分担ができているか。
- 読ませるべきカードと、状態フィードバックだけの演出が混ざって見えていないか。

辛口で見てほしい観点:

- 証拠取得時に「カードを手に入れた」快感があるか。
- ただのDOM通知に見えていないか。
- 現地調査の写真タップから証拠カード出現までの一連の流れに手触りがあるか。

### C. ノベルADVとしての統一感

確認してほしいこと:

- タイトル、案件選択、依頼受付、現地調査、資料照合、鑑定判断、報告対決が同じゲームの画面としてつながっているか。
- 案件選択だけが強く、以降のフェーズが再び業務フォームに戻って見えないか。
- 主人公の新人鑑定士、先輩鑑定士、依頼者の存在感は十分か。
- 主人公の台詞が入ったことで、プレイヤーの分身として機能しているか。
- 依頼者圧力や先輩レビューがドラマとして伝わるか。

辛口で見てほしい観点:

- 「ノベルゲーム調」と言いながら、実態は説明パネルが増えただけではないか。
- 台詞が教材説明に寄りすぎて、キャラクターの声になっていないか。
- 会話スキップ時にゲーム理解が崩れないか。

### D. ゲーム画面内完結度

確認してほしいこと:

- 全ての操作がゲーム画面内に収まって見えるか。
- Webページの下にフォームが続くような印象が残っていないか。
- サイドの証拠ボード、フェーズ目標、スコア、会話、本体操作が競合していないか。
- 画面密度が高すぎないか。
- モバイル幅で縦スクロールが長すぎないか。

### E. アクセシビリティと低刺激

確認してほしいこと:

- `prefers-reduced-motion` と低刺激モードで、案件選択アニメやカード演出が過剰に残っていないか。
- 低刺激時に演出を省略しても、何が起きたか情報が失われていないか。
- ノベル本文の `aria-live`、進行ボタンの `aria-label`、フェーズ遷移後のフォーカス移動は十分か。
- スキップ操作がキーボードとスクリーンリーダーで破綻していないか。
