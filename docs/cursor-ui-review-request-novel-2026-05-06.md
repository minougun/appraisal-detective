## 直近の主要変更

### 1. ノベルADV風シーンの追加

`app.js` に以下の構造を追加した。

- `state.storyIndex`
- `phaseStoryLines()`
- `novelSceneMarkup()`
- `bindNovelScene()`

案件選択、依頼受付、現地調査、資料照合、鑑定判断、報告対決、結果画面の各タイミングで `novelSceneMarkup()` を表示し、背景画像・人物表示・台詞ボックス・操作ボタンで物語の進行感を出している。

### 2. ボタン文言変更

ノベルシーンの送りボタンを以下のように変更した。

- 旧: `次の台詞`
- 新: `次へ`
- 最終台詞到達後: `確認済み`

### 3. メッセージスキップ機能

ノベルシーンに `スキップ` ボタンを追加した。

- `data-novel-skip` で実装
- 押すと現在フェーズの最後の台詞へ移動
- 最終台詞では disabled
- スキップ時に先輩メモへ「会話をスキップした。最後の台詞だけ確認して判断へ進める。」を表示

### 4. ノベルUI用CSS

`styles.css` に以下の系統を追加した。

- `.novel-scene`
- `.novel-backdrop`
- `.novel-cast`
- `.novel-character`
- `.novel-box`
- `.novel-controls`
- `.novel-control-buttons`
- `.novel-next`
- `.novel-skip`

デスクトップでは背景・人物・台詞箱が同一画面に見える構成、モバイルでは人物と台詞が縦に積まれる構成にしている。

### 5. E2Eテスト追加

`tests/appraisal-detective-flow.spec.js` に以下を追加/更新した。

## UIレビューで特に見てほしい論点

### A. ノベルゲーム調としての成立度

- 背景・人物・台詞箱の組み合わせが、ゲームの第一印象を強めているか
- 以前の「業務ツール感」「SaaS感」から十分に離れられているか
- 机上捜査、鑑定事務所、事件ファイルという方向性とノベルADV演出が噛み合っているか
- 人物表示が簡素すぎて、逆に安っぽく見えないか

### B. 画面構成と情報密度

- ノベルシーンが目立ちすぎて、肝心の案件選択や操作UIを圧迫していないか
- 逆に、ノベルシーンが弱すぎて「後付け感」になっていないか
- サイドの証拠ボード、フェーズ目標、スコア、ノベルシーンが競合していないか
- 初見ユーザーが「今どこを読んで、次に何を押すか」を迷わないか

### C. 「次へ」「スキップ」の導線

- `次へ` という文言は自然か
- 最終台詞の `確認済み` はわかりやすいか、それとも `調査へ進む` など行動名に寄せるべきか
- `スキップ` の位置、見た目、disabled状態は自然か
- スキップすると物語理解が破綻しないか
- スキップ機能があることで、カジュアル層のテンポ改善につながっているか

### D. 鑑定士追体験との両立

- 専門用語や鑑定判断の重さが、ノベル演出によって薄まっていないか
- 依頼者圧力、先輩鑑定士の助言、証拠提示がドラマとして立っているか
- 「専門職シム」と「推理ADV」のどちらにも中途半端になっていないか

### E. モバイル/レスポンシブ

- スマホ幅でノベルシーン、台詞、操作ボタンが読みやすいか
- ボタンの押しやすさ、文字の折り返し、縦スクロール量に問題がないか
- 横長前提のADV画面を無理にスマホへ押し込んでいないか

### F. アクセシビリティ/低刺激

- `次へ` / `スキップ` がキーボード操作とスクリーンリーダーで理解しやすいか
- `prefers-reduced-motion` と低刺激モードの方針と矛盾していないか
- 会話進行時の情報が視覚だけに依存していないか
- disabled状態やフォーカス移動に問題がないか
