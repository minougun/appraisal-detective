# 鑑定探偵 商用公開前 人間レビュー用チェックリスト

作成日: 2026-05-07

## 目的

自動テストと仕様充足ゲートでは測れない、商用ゲームとしての体験品質を確認する。

`npm run test:persona` の `persona_average=100.0` は仕様充足ゲートであり、商用品質の平均点ではない。商用95点判定は、このチェックリストに基づく人間プレイテストで別途判断する。

## 対象

- URL: `http://127.0.0.1:44561/`
- ローカルパス: `/mnt/c/Users/minou/appraisal-detective/`
- 重点案件:
  - 001 川辺町住宅地 / イージー
  - 002 駅前商業地 / ノーマル
  - 003 南口再開発予定地 / ハード

## 合格条件

- 案件001を初見ユーザーが15分以内に完了できる。
- 案件001で「価格形成要因」「最有効使用」「取引事例比較法」の意味を、ゲーム内文脈で最低限説明できる。
- 案件002で「NOI」「DCR」「収益還元法」が、単なる用語暗記ではなく判断に使われたと感じられる。
- 案件003で「最有効使用」は全鑑定の前提であり、再開発予定地では用途・規模判断が中心になることが伝わる。
- 報告対決で、証拠提示、依頼者反論、再反論の流れに緊張感がある。
- 低刺激ONで、音、揺れ、急な点滅、圧力フラッシュが不快にならない。
- 5人レビュー平均が95点以上、かつ80点未満のペルソナがない。

## ペルソナ別確認

### P1 ADV・捜査読み好き

- 事件ファイル選択から現地調査まで、ゲームに入った感があるか。
- 新人鑑定士、先輩、依頼者が説明役だけに見えないか。
- 依頼者の反論に対して、証拠で返す快感があるか。
- 2周目で別の台詞や別の判断を見たいと思うか。

### P2 週1カジュアル

- 案件001で次に何をすればよいか迷わないか。
- 専門用語の説明量が多すぎないか。
- チュートリアルが邪魔ではなく、判断の助けになっているか。
- スキップしてもゲームの流れを失わないか。

### P3 鑑定プロシマ

- 価格形成要因、対象不動産の確定、事情補正、時点修正、最有効使用の扱いに違和感がないか。
- 案件001〜003の最有効使用マトリクスが、鑑定判断の前提として機能しているか。
- 試算価格の調整幅、検算、監査コメントが専門職シムとして薄すぎないか。
- 用語がラベルではなく、判断の根拠として使われているか。

### P4 感覚過敏・配慮ユーザー

- 低刺激ONでBGM/SEが停止し、圧フラッシュや証拠ボード演出が抑えられるか。
- キーボードだけで案件001を完走できるか。
- フォーカス位置がフェーズ切替後に迷子にならないか。
- 色だけで重要状態を判別させていないか。

### P5 リプレイ・スコア厨

- 通常レビューと監査レビューの違いが分かるか。
- 市場シナリオ差、証拠提示順、再反論、調整幅でスコアを詰めたくなるか。
- 記録一覧から次周目標が見えるか。
- 固定正解の暗記だけで終わらないか。

## ストア素材確認

- タイトル画面が16:9スクリーンショットとして成立する。
- 事件ファイル選択のスクリーンショットが、ゲームの顔として使える。
- 現地調査で証拠カードが出る瞬間が映える。
- 資料照合で「矛盾を見つける」画面が伝わる。
- 報告対決で依頼者反論と証拠提示が一枚絵として伝わる。
- 最終レビュー画面で、スコア、監査コメント、次周目標が見える。

## アセット・権利確認

- `assets-manifest.json` の全生成画像について、用途、生成日、モデル、プロンプトが記録されている。
- `docs/third-party-audio-notices.md` にBGM/SEの出所、ライセンス、クレジットが記録されている。
- ゲーム内クレジットと配布ページに、画像、音源、フォント、生成AI利用の表示方針がある。
- 商用公開先の規約に反する素材がない。
