# VOICEVOX ADV会話音声ノウハウ

作成日: 2026-05-14  
対象: ADV / ノベル / 会話主体ゲームで、VOICEVOXなどの合成音声をキャラクター会話に入れるときの設計メモ。

このメモは「鑑定DE探偵」での調整から得た知見を、ほかのADVゲームにも流用できるように整理したもの。結論から言うと、自然さは speaker 選びだけでは決まらない。**声質、台詞、間、役割分担、fallback、実聴チェック**をセットで見る必要がある。

## 基本方針

1. **speakerより先に、台詞の役割を決める。**
   同じ声でも、台詞が説明文なら「AI案内音声」に聞こえる。キャラクターに聞かせたいなら、その人物がその場で考えた言葉にする。

2. **ナレーター、主人公、先輩、依頼者を同じ文体にしない。**
   声が違っても、全員がチュートリアル文を読むと混ざる。役割ごとに文の作りを変える。

3. **若い声を使うときほど、fallbackを厳しくする。**
   若さを出す speaker は、少しの台詞ミスで軽い、幼い、アホっぽい印象になりやすい。第一候補が無いときに別の若い声へ自動fallbackするより、代替音声か無音に落とすほうが安全な場合がある。

4. **パラメータ調整は最後の補正。**
   `pitch` や `intonation` で機械音声っぽさを消そうとしても、台詞が案内文なら「抑揚のある案内音声」になるだけ。

5. **テスト通過と自然さ判定を分ける。**
   E2Eは「VOICEVOXが呼ばれる」「speakerが選ばれる」「fallbackしない」を保証できる。自然に聞こえるかは、実際のChrome再生を耳で確認する。

## 役割別の作り方

### ナレーター

目的: 状況を置く。感情を説明しすぎない。

向いている文:

```txt
相続協議の依頼。
川辺町住宅地の依頼ファイルを開いた。
価格時点の市場条件を確認する。
```

避ける文:

```txt
あなたは新人鑑定士として、これから依頼者の事情を聞きながら正しい評価額を導き出します。
```

長いナレーションは「聞かされている」感じが出る。短文なら速度を上げてもよいが、長文はrate capを下げる。

推奨:

```js
narrator: {
  speed: 1.0,
  maxRate: 1.25,        // 短文だけ軽快にする
  longTextRateCap: 1.05 // 長文は速くしすぎない
}
```

### 主人公 / プレイヤー分身

目的: 操作の説明ではなく、判断の途中経過を言う。

よい形:

```txt
依頼ファイルを開く。まず、どの根拠が使えるか見る。
事情は聞く。でも、評価額に入れる根拠は分ける。
収入と費用を見ないまま、利回りだけは動かせない。
```

避ける形:

```txt
依頼ファイルを開きます。まずは、価格を支える根拠から見ます。
事情は聞きます。ただ、評価額は根拠から組み立てます。
次に収入と費用の内訳を確認します。
```

理由:

- 敬体は依頼者への返答には使えるが、内心に使うと案内音声っぽい。
- 「組み立てます」「確認します」はUI操作説明に寄る。
- 「使える根拠」「評価額に入れる根拠」のように、ゲーム操作と接続する語のほうが自然。

依頼者へ直接返すときだけ敬体にする。

```txt
内心:
事情は聞く。でも、評価額に入れる根拠は分ける。

依頼者向け:
ご事情は聞きます。ただ、評価額に入れる根拠は分けます。
```

### 先輩 / メンター

目的: 視点を戻す。長く講義しない。

よい形:

```txt
そこだ。希望は聞く。価格には混ぜるな。
だから、希望から決めない。先に収益の中身だ。
正常価格と担保リスクは、同じ判断に混ぜない。
```

避ける形:

```txt
探そう。……いいな。
選別しよう。……急がなくていい。
返そう。声を荒げる必要はない。
```

理由:

- 命令をただ柔らかくすると、今度はワークショップ講師のようになる。
- `……` は演技の間になりやすく、メンターが大げさに聞こえる。
- 先輩は短く戻す。説明は主人公か資料に分ける。

### 依頼者 / クライアント

目的: 都合、不安、圧をにじませる。理論を理解しすぎない。

よい形:

```txt
妹に見せる額としては、低い額だと困るんです。
利回りを少し動かすだけで、そこまで価格に出るんですね。
市が前向きでも足りないんですか。……接道で、そこまで変わるんですね。
```

避ける形:

```txt
つまり正常価格と担保リスクを分け、収益価格の前提を精査する必要があるということですね。
```

依頼者は鑑定理論を即理解しすぎると不自然になる。理解は半歩だけ進める。

## Speaker設計

### 固定すべき声は固定する

主人公の印象が作品全体に効く場合、fallback候補を広げすぎない。

例:

```js
player: {
  speakerPreferences: ["雀松朱司"],
  strictSpeakerPreference: true,
  fallbackPolicy: {
    ifPreferredSpeakerMissing: "browserFallbackOrSilent",
    avoidSpeakers: ["白上虎太郎", "玄野武宏"]
  }
}
```

理由:

- `白上虎太郎` は若さが出るが、軽さ・幼さ・アホっぽさに寄る危険がある。
- `玄野武宏` は低めで落ち着くが、依頼者男性にも使うと混線しやすい。
- 作品の第一声を担う主人公は、自動fallbackで印象が変わると危険。

### 混線を避ける

同じ声質の男性を複数役に置くと、字幕なしで分かりにくい。

役割の分離例:

```txt
ナレーター: 落ち着いた女性 / 中性的な声
主人公: 若い男性、軽すぎない
先輩: 低め、短く補正する
依頼者: 案件ごとに不安・圧・年齢感を変える
```

声だけで分けきれないときは、文体で分ける。

```txt
主人公: 判断・仮説・気づき
先輩: 視点の補正
ナレーター: 状況
依頼者: 都合・不安・圧
```

## パラメータ目安

### 若いが賢そうな男性主人公

```js
defaults: {
  speed: 1.00,
  pitch: -0.008,
  intonation: 1.02,
  volume: 0.9,
  pre: 0.10,
  post: 0.14,
  maxRate: 1.03
}
```

狙い:

- speedは下げすぎない。遅いと鈍く聞こえる。
- pitchは少しだけ低め。下げすぎると新人感が消える。
- intonationは少し残す。下げすぎると平板な案内音声になる。
- postを少し長めにして、文末が詰まる感じを避ける。

避ける設定:

```js
intonation: 0.94
post: 0.08
```

これは「賢そう」ではなく「抑揚を殺した若手案内役」になりやすい。

### 先輩

```js
defaults: {
  speed: 0.98,
  pitch: -0.025,
  intonation: 0.98,
  volume: 0.96,
  pre: 0.14,
  post: 0.16,
  maxRate: 1.00
}
```

低くしすぎるより、台詞を短くするほうが効く。

### ナレーター

```js
defaults: {
  speed: 1.00,
  pitch: -0.025,
  intonation: 1.08,
  volume: 0.9,
  pre: 0.10,
  post: 0.10,
  maxRate: 1.25
}
```

短文ナレーションだけ1.25倍にする。説明長文は上限を落とす。

## 台詞の自然化ルール

### 1. 説明文を思考文にする

```txt
説明文:
まずは、価格を支える根拠から見ます。

思考文:
まず、どの根拠が使えるか見る。
```

### 2. 抽象語を操作語に寄せる

```txt
硬い:
価格を支える根拠

自然:
使える根拠
評価額に入れる根拠
比べられる資料
```

### 3. 内心は常体、対人は敬体

```txt
内心:
利回りだけは動かせない。

依頼者向け:
利回りだけは動かせません。
```

### 4. 専門説明は一文に詰めない

悪い例:

```txt
利回りを下げると価格は高く見えるため、希望から決めず、先に収入と費用を確認する必要があります。
```

よい例:

```txt
新人「利回りを下げると、同じ収益でも価格は高く見えます。」
先輩「だから、希望から決めない。先に収益の中身だ。」
```

### 5. `……` は使いすぎない

`……` はVOICEVOXで強い演技の間になることがある。依頼者のためらいには有効だが、メンターや主人公の通常判断には使いすぎない。

使ってよい例:

```txt
市が前向きでも足りないんですか。……接道で、そこまで変わるんですね。
```

避ける例:

```txt
探そう。……いいな。
```

## Fallback設計

### 危険なfallbackを自動採用しない

本命speakerがいないときに、近い年齢のspeakerへ落とすと、キャラ印象が壊れることがある。

推奨:

```js
function shouldEnablePlayerVoicevox(availableSpeakers) {
  return availableSpeakers.some((speaker) => speaker.name === "雀松朱司");
}
```

speakerが無い場合:

```txt
VOICEVOX player voice unavailable
ブラウザ代替音声へfallback
または無音
```

避ける:

```txt
雀松朱司が無いので白上虎太郎へ自動fallback
```

### テストで固定する

確認したいこと:

```txt
雀松朱司がある -> speaker 52 など本命IDで audio_query
雀松朱司がない -> 白上虎太郎のspeaker IDへ audio_query しない
```

## 実装チェックリスト

1. speaker一覧を取得する。
2. 役ごとの第一候補speakerを決める。
3. 混線するspeakerを `avoidSpeakers` として明記する。
4. 主人公や重要キャラは `strictSpeakerPreference` を検討する。
5. 台詞を「誰の言葉か」で分類する。
6. 内心、依頼者向け、ナレーション、メンター補正を混ぜない。
7. 短文と長文でrate capを分ける。
8. fallback時のステータス表示を分かりやすくする。
9. 低刺激モードで再生中音声とqueueが止まることを確認する。
10. 実機Chromeで `VOICE VOICEVOX` 表示のまま聞く。

## 実聴チェックリスト

必ずブラウザ上の実再生で確認する。生成WAVだけでは、UI遷移、遅延、重なり、キャンセル挙動が分からない。

確認順:

1. タイトル直後の第一声
2. 主人公の内心
3. 依頼者への敬体返答
4. 先輩との連続再生
5. 依頼者との連続再生
6. ナレーターとの連続再生
7. 長文説明
8. `……` を含む台詞
9. speaker欠落時のfallback
10. 低刺激モードON時の即停止

判定基準:

```txt
若いが軽すぎない
賢そうだが平板ではない
案内音声ではなく、人物の言葉に聞こえる
先輩と主人公が混ざらない
依頼者が理論を理解しすぎない
長文が講義に聞こえない
VOICEVOXが無い環境で不自然な別声に落ちない
```

## よくある失敗

### 失敗1: 声だけ変えて台詞を直さない

```txt
依頼ファイルを開きます。まずは、価格を支える根拠から見ます。
```

この文は誰が読んでも案内音声に寄る。

修正:

```txt
依頼ファイルを開く。まず、どの根拠が使えるか見る。
```

### 失敗2: 若い声を使えば主人公らしくなると思う

若い声は、台詞が少し説明的なだけで軽く聞こえる。若さより「判断している感じ」を優先する。

### 失敗3: 抑揚を下げすぎる

賢そうにしたくて `intonation` を下げると、平板な読み上げになる。賢さはパラメータより台詞で作る。

### 失敗4: メンターを柔らかくしすぎる

命令を避けるのはよいが、柔らかい補足を足しすぎると講師になる。

```txt
弱い:
選別しよう。……急がなくていい。

よい:
使える根拠だけを選ぼう。
```

### 失敗5: テスト結果を自然さの証明にする

`persona_average=100.0` やE2E通過は、統合品質の証明であって、聴感品質の証明ではない。音声自然さは必ず実聴で判定する。

## 推奨テンプレート

```js
const voiceCast = {
  narrator: {
    speakerPreferences: ["九州そら"],
    defaults: { speed: 1.0, pitch: -0.025, intonation: 1.08, volume: 0.9, pre: 0.1, post: 0.1, maxRate: 1.25 },
  },
  player: {
    speakerPreferences: ["雀松朱司"],
    strictSpeakerPreference: true,
    defaults: { speed: 1.0, pitch: -0.008, intonation: 1.02, volume: 0.9, pre: 0.1, post: 0.14, maxRate: 1.03 },
  },
  mentor: {
    speakerPreferences: ["青山龍星"],
    defaults: { speed: 0.98, pitch: -0.025, intonation: 0.98, volume: 0.96, pre: 0.14, post: 0.16, maxRate: 1.0 },
  },
};
```

台詞テンプレート:

```txt
ナレーター:
{状況を一文で置く。}

主人公:
{判断する。常体。短く。}

依頼者:
{都合・不安・圧を出す。理論は理解しすぎない。}

先輩:
{視点を戻す。短く。講義しない。}
```

## 最終判断ルール

本番反映してよい条件:

```txt
E2Eが通る
VOICEVOX speaker選択が固定されている
危険fallbackがない
低刺激停止が効く
Chrome上で実聴済み
第一声が案内音声に聞こえない
主人公と先輩と依頼者が聞き分けられる
```

本番不可の条件:

```txt
実聴していない
VOICE 代替音声でしか確認していない
本命speaker欠落時に軽い別speakerへ落ちる
主人公台詞が敬体の案内文になっている
長文説明が講義に聞こえる
```

結論:

**ADV会話音声の自然さは、声の設定ではなく脚本設計から作る。**  
VOICEVOXのパラメータは、よい台詞を少し整えるための補正として使う。
